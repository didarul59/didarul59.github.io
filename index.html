MnO Wannier90 Tutorial: Complete Workflow Analysis

1. Introduction

This document summarizes the successful Wannier90 calculation for MnO (Manganese Oxide), an antiferromagnetic insulator with rock-salt structure. Both up and down spin channels ran successfully with identical input files, providing a reference for more complex materials like FeO.

2. Workflow Steps

Step 1: Generate k-points for NSCF
~/q-e-qe-7.3/external/wannier90/utility/kmesh.pl 4 4 4 >> MnO_nscf.in
Step 2: Run NSCF Calculation
mpirun -np 4 pw.x -i MnO_nscf.in > MnO_nscf.out
Step 3: Generate k-points for Wannier90
~/q-e-qe-7.3/external/wannier90/utility/kmesh.pl 4 4 4 wan >> MnO_up.win ~/q-e-qe-7.3/external/wannier90/utility/kmesh.pl 4 4 4 wan >> MnO_down.win
Step 4: Wannier90 Pre-processing
wannier90.x -pp MnO_up wannier90.x -pp MnO_down
Step 5: Run pw2wannier90
pw2wannier90.x < pw2wan_MnO_up.in > pw2wan_MnO_up.out pw2wannier90.x < pw2wan_MnO_down.in > pw2wan_MnO_down.out
Step 6: Run Wannier90
wannier90.x MnO_up wannier90.x MnO_down
Step 7: Verify Success
# Check for convergence grep "Final State" MnO_up.wout grep "Spread" MnO_up.wout # Plot bands gnuplot MnO_up_band.gnu

3.2 Band Calculation Logic
Total valence electrons = 26
Spin-polarized (nspin=2) → each band holds 1 electron
Valence bands needed = 26 bands
Extra empty bands for accuracy = 10 bands
Total nbnd = 36
3.3 Relationship with Wannier90 Parameters
nbnd (36) ≥ num_bands (26) ≥ num_wann (18) + exclude_bands (10)
  • nbnd = 36: What pw.x calculates (includes empty bands)
  • num_bands = 26: What Wannier90 reads from DFT (valence bands)
  • exclude_bands : 1-10: Removes core states
  • num_wann = 18: Final Wannier functions

5. Key Insights

5.1 Identical .win Files for Both Channels
  • MnO's symmetric band structure allows the same parameters for up and down
  • Both channels ran successfully without modification
5.2 Relationship Between nbnd and Wannier Parameters
Parameter Value Meaning
nbnd 36 Total bands calculated by pw.x
num_bands 26 Bands used by Wannier90
exclude_bands 1-10 Core states excluded
num_wann 18 Target Wannier functions
Constraint: nbnd ≥ num_bands ≥ num_wann + excluded_bands
5.3 Projections Count
Mn:s;d → 1s + 5d = 6 orbitals per Mn × 2 Mn = 12
O:p → 3 orbitals per O × 2 O = 6
Total = 18 Wannier functions
5.4 Energy Windows
  • Outer window (5-35 eV): Contains all relevant bands
  • Frozen window (5-18 eV): Inner window for disentanglement
  • Works because MnO has a clean band gap
5.5 Why FeO is Different

Unlike MnO, FeO shows:

  • Asymmetric band structures between up and down channels
  • Deep core states in down channel (-72 eV to -38 eV)
  • Different total band counts (72 vs 60)
  • Requires channel-specific parameters

6. Troubleshooting Guide

6.1 Common Errors and Solutions
Error Cause Solution
mismatch in .eig file num_bands doesn't match DFT Check actual bands: awk '{print $1}' FeO_up.eig | sort -nu | tail -1
fewer states in frozen window Not enough bands in energy range Widen energy windows or use frozen_bands
more states in frozen window Too many bands in range Narrow window or use exact frozen_bands
k-point mismatch K-points differ between files Use kmesh.pl for consistency

GPU-Accelerated Quantum Espresso: Complete Setup Guide

Author: Md. Didarul Islam

System: Arch Linux | NVIDIA GPU | CUDA 12.2 | NVIDIA HPC SDK 23.9

QE Version: 7.3

A comprehensive guide to building and running GPU-accelerated Quantum Espresso with CUDA support for high-performance DFT calculations.

📋 Step 1: Prerequisites

Required Software
  • CUDA Toolkit 12.2
  • NVIDIA HPC SDK 23.9
  • OpenMPI (compatible with NVIDIA HPC SDK)
  • SCALAPACK libraries
Verify Your GPU

Make sure your NVIDIA GPU is properly detected before proceeding:

nvidia-smi

⚙️ Step 2: Configure Environment

Set up the necessary environment variables for QE GPU support. Add the following to your ~/.bashrc for a permanent configuration:

echo '' >> ~/.bashrc echo '# QE GPU Environment' >> ~/.bashrc echo 'export NVHPC_HOME=/opt/nvidia/hpc_sdk/Linux_x86_64/23.9' >> ~/.bashrc echo 'export CUDA_HOME=/opt/nvidia/hpc_sdk/Linux_x86_64/23.9/cuda/12.2' >> ~/.bashrc echo 'export PATH=$NVHPC_HOME/compilers/bin:$CUDA_HOME/bin:$PATH' >> ~/.bashrc echo 'export LD_LIBRARY_PATH=$CUDA_HOME/lib64:$NVHPC_HOME/compilers/lib:$NVHPC_HOME/REDIST/compilers/lib:/opt/nvidia/hpc_sdk/Linux_x86_64/23.9/comm_libs/openmpi4/lib:$LD_LIBRARY_PATH' >> ~/.bashrc echo 'export PATH=/opt/nvidia/hpc_sdk/Linux_x86_64/23.9/comm_libs/openmpi4/bin:$PATH' >> ~/.bashrc echo 'export MPC_GDR_SYNC_INIT=1' >> ~/.bashrc echo 'export CUDA_MANAGED_FORCE_DEVICE_ALLOC=1' >> ~/.bashrc echo 'export CUDA_VISIBLE_DEVICES=0,1' >> ~/.bashrc echo 'export OMP_NUM_THREADS=1' >> ~/.bashrc echo 'export OMPI_MCA_btl=^openib' >> ~/.bashrc

Then reload your shell:

source ~/.bashrc

🔨 Step 3: Build Quantum Espresso with GPU Support

Navigate to QE Directory
cd /home/cuda/q-e-qe-7.3
Configure with GPU Support
./configure \ --with-cuda=/opt/nvidia/hpc_sdk/Linux_x86_64/23.9/cuda/12.2 \ --with-cuda-runtime=12.2 \ --with-cuda-cc=61 \ --enable-openmp \ --with-scalapack=yes \ --with-cuda-mpi=yes \ FC=nvfortran \ MPIF90=mpif90 \ CC=nvc \ CXX=nvc++

Configuration Options Explained:

  • --with-cuda: Path to CUDA toolkit
  • --with-cuda-runtime=12.2: CUDA runtime version
  • --with-cuda-cc=61: Compute capability (6.1 for GTX 1050 Ti, 6.1 for GTX 1060 — adjust for your GPU)
  • --enable-openmp: Enable OpenMP parallelization
  • --with-scalapack=yes: Enable ScaLAPACK for distributed memory
  • --with-cuda-mpi=yes: Enable CUDA-aware MPI
  • FC=nvfortran: NVIDIA Fortran compiler
  • MPIF90=mpif90: MPI Fortran wrapper
  • CC=nvc: NVIDIA C compiler
  • CXX=nvc++: NVIDIA C++ compiler
Build QE
make all

This will compile all QE executables with GPU acceleration. The build may take 10–20 minutes depending on your system.

Add QE Binary Path to .bashrc

After building, add the QE bin directory to your PATH so the executables are accessible from anywhere:

echo 'export PATH="/home/cuda/q-e-qe-7.3/bin:$PATH"' >> ~/.bashrc source ~/.bashrc

💡 Note: The path /home/cuda/q-e-qe-7.3/bin may vary depending on where you extracted and built QE. Adjust the path to match your QE installation directory.

🚀 Step 4: Run GPU-Accelerated QE

Once everything is set up, run your QE calculations with GPU acceleration:

mpirun -np 2 pw.x -i scf.in > scf.out

📊 Step 5: Monitor GPU Usage

Keep an eye on GPU utilization and memory during your QE runs:

# Real-time monitoring nvidia-smi # Watch mode (update every 1 second) watch -n 1 nvidia-smi

⚡ Step 6: Performance Tips

  • Use -npool to distribute k-points across GPUs
  • Set OMP_NUM_THREADS=1 for MPI-only parallelization
  • Match -np to the number of GPUs for best performance
  • Monitor GPU utilization with nvidia-smi during runs
  • Use exclusive process mode to prevent GPU sharing conflicts

✅ Conclusion

This setup provides a complete workflow for GPU-accelerated Quantum Espresso calculations. With the environment configured and QE built with GPU support, you can now leverage NVIDIA GPUs for high-performance DFT calculations.

Author: Didarul Islam

System: Arch Linux | NVIDIA GTX 1050 Ti | CUDA 12.2

A step-by-step tutorial for building and running the GPU-accelerated version of VAMPIRE micromagnetic simulation code on Linux systems.

Introduction

VAMPIRE (Vampire is a highly extensible atomistic magnetic simulation package) is a powerful code for micromagnetic simulations. GPU acceleration can provide up to 20x speedup for large-scale simulations.

This tutorial provides a single script to build VAMPIRE with CUDA support on modern Linux systems.

Prerequisites

Software
  • NVIDIA Drivers (version ≥ 535)
  • CUDA Toolkit (version 12.2 or compatible)
  • Git

Check your GPU:

nvidia-smi
